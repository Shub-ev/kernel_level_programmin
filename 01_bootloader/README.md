# Bootable Floppy Image Build System

## Overview

This project demonstrates how to use **NASM** and **Make** to build a bootable floppy disk image from a raw x86 assembly source file.

The build process performs the following steps:

1. Assemble `main.asm` into a raw binary (`main.bin`).
2. Copy the binary into a floppy image (`main.img`).
3. Expand the image to the standard floppy disk size of 1.44 MB.
4. Optionally clean all generated files.

---

# Project Structure

```
project/
│
├── src/
│   └── main.asm
│
├── build/
│   ├── main.bin
│   └── main.img
│
└── Makefile
```

## Directory Description

### src/

Contains all assembly source files.

Example:

```
src/main.asm
```

This file contains the bootloader or operating system code written in x86 assembly.

---

### build/

Contains generated files.

#### main.bin

Raw binary output generated by NASM.

```
main.asm
   ↓
NASM
   ↓
main.bin
```

This file contains the machine code that the CPU executes.

---

#### main.img

Bootable floppy disk image.

```
main.bin
   ↓
copy
   ↓
main.img
```

The image is expanded to 1.44 MB and can be booted in emulators such as QEMU, Bochs, or VirtualBox.

# Makefile Explanation

The Makefile automates the build process.

## Variables

```make
ASM=nasm

SRC_DIR=src
BUILD_DIR=build
```

### ASM

Specifies the assembler to use.

```make
ASM=nasm
```

Equivalent to:

```bash
nasm
```

---

### SRC_DIR

Stores the location of source files.

```make
SRC_DIR=src
```

---

### BUILD_DIR

Stores the location of generated files.

```make
BUILD_DIR=build
```

---

# Building the Floppy Image

```make
$(BUILD_DIR)/main.img: $(BUILD_DIR)/main.bin
	cp $(BUILD_DIR)/main.bin $(BUILD_DIR)/main.img
	truncate -s 1440k $(BUILD_DIR)/main.img
```

## Target

```make
build/main.img
```

This target depends on:

```make
build/main.bin
```

Meaning:

```
main.img cannot be created until main.bin exists.
```

---

## Copy Operation

```make
cp $(BUILD_DIR)/main.bin $(BUILD_DIR)/main.img
```

Equivalent command:

```bash
cp build/main.bin build/main.img
```

Creates a floppy image from the binary.

Result:

```
main.bin
   ↓
copy
   ↓
main.img
```

---

## Expand Image Size

```make
truncate -s 1440k $(BUILD_DIR)/main.img
```

Expands the image to:

```
1440 KB
=
1.44 MB
```

which is the standard size of a floppy disk.

After expansion:

```
main.img
┌───────────────────────┐
│ Bootloader Code       │
│ Empty Space           │
│ Empty Space           │
│ Empty Space           │
└───────────────────────┘
```

---

# Building the Binary

```make
$(BUILD_DIR)/main.bin: $(SRC_DIR)/main.asm
	$(ASM) $(SRC_DIR)/main.asm -f bin -o $(BUILD_DIR)/main.bin
```

## Target

```make
build/main.bin
```

Depends on:

```make
src/main.asm
```

Meaning:

```
Whenever main.asm changes,
main.bin must be rebuilt.
```

---

## NASM Command

Generated command:

```bash
nasm src/main.asm -f bin -o build/main.bin
```

### Parameters

#### src/main.asm

Input assembly file.

---

#### -f bin

Output format.

```bash
-f bin
```

Generates a raw binary file.

No ELF headers.

No PE headers.

No executable metadata.

Only machine code.

Example:

```
B8 00 00
CD 10
EB FE
```

---

#### -o build/main.bin

Output filename.

Generated file:

```bash
build/main.bin
```

---

# Dependency Flow

```
src/main.asm
       │
       ▼
build/main.bin
       │
       ▼
build/main.img
```

Make automatically follows these dependencies.

---

# Clean Target

```make
clean:
	rm $(BUILD_DIR)/*
```

Removes all generated files.

Equivalent command:

```bash
rm build/*
```

Example:

Before:

```
build/
├── main.bin
└── main.img
```

After:

```
build/
```

(empty)

---

# Build Process Step-by-Step

When you run:

```bash
make
```

Make performs the following:

### Step 1

Check whether:

```
build/main.img
```

exists.

---

### Step 2

If not, check its dependency:

```
build/main.bin
```

---

### Step 3

If `main.bin` does not exist:

```bash
nasm src/main.asm -f bin -o build/main.bin
```

---

### Step 4

Create image:

```bash
cp build/main.bin build/main.img
```

---

### Step 5

Expand image:

```bash
truncate -s 1440k build/main.img
```

---

### Step 6

Build complete.

Generated files:

```
build/
├── main.bin
└── main.img
```

---

# Running the Image

Example using QEMU:

```bash
qemu-system-i386 -fda build/main.img
```

QEMU loads:

```
main.img
   ↓
Boot Sector
   ↓
CPU starts executing
```
